# Interactive UMAP Explorer

Dash-based web application for interactive UMAP visualization of single-cell RNA-seq retinal development data. Supports 11 GEO (GSE) datasets with three exploration modes: per-dataset analysis, cross-dataset comparison, and an integrated Harmony-corrected shared UMAP space.

## Pages

| Page | Path | Description |
|------|------|-------------|
| **Single Dataset** | `/` | Explore one dataset at a time — color by cell type, gene expression, QC metrics, or custom Leiden clusters. Includes violin plots for expression distributions. |
| **Compare** | `/comparison` | Side-by-side UMAPs from two independently selected datasets with linked coloring and gene search. |
| **Integrated** | `/integrated` | All 11 datasets projected into a shared UMAP space via Harmony integration. View global cell-type relationships across species and developmental stages. |

## Architecture

```
app.py                          # Dash entry point — init, global layout, page registration, WSGI export
├── pages/
│   ├── single_dataset.py       # Landing page (/) — callbacks, controls, gene search, violin plots
│   ├── comparison.py           # /comparison — side-by-side UMAPs, independent dataset selectors
│   └── integrated.py           # /integrated — Harmony-corrected multi-dataset view
├── components/
│   ├── data_loader.py          # Parquet loading with PyArrow memory mapping + module-level cache
│   ├── umap_figure.py          # Core UMAP rendering — Scattergl (2D) / Scatter3d (3D), adaptive sampling
│   ├── controls.py             # Sidebar — dataset dropdown, coloring selector, gene search trigger
│   ├── gene_selector.py        # Gene search / autocomplete UI
│   ├── info_panel.py           # Dataset metadata and citation display
│   └── violin_plot.py          # Expression distribution violin plots
├── preprocessing/
│   ├── build_data_cache.py     # Extract parquet from 04_clustered.h5ad files
│   ├── build_integrated.py     # Harmony integration → shared UMAP space
│   ├── extract_gene_metadata.py # Build gene_index.json for autocomplete
│   └── validate_cache.py       # Validate parquet: NaN, bounds, column coverage
├── assets/
│   └── style.css               # Single stylesheet (served automatically by Dash)
├── data/
│   ├── dataset_registry.json   # Dataset metadata (committed)
│   ├── gene_index.json         # Gene search index (committed)
│   └── *.parquet / *.h5ad      # Cached data (gitignored, generated by preprocessing)
├── Dockerfile                  # Production image — python:3.11-slim, gunicorn 4 workers, port 8050
├── nginx.conf                  # Reverse proxy — WebSocket upgrade, 50M body limit, asset caching
├── start.sh                    # Local dev launcher
└── requirements.txt
```

## Quick Start (Development)

**Prerequisites**: Python 3.11+, virtual environment, pre-built data in `data/`.

```bash
# 1. Clone and set up environment
python3 -m venv ../.venv
source ../.venv/bin/activate
pip install -r requirements.txt

# 2. Build data cache (requires 04_clustered.h5ad files from an external directory)
python preprocessing/build_data_cache.py --data-dir /path/to/h5ad_files
python preprocessing/extract_gene_metadata.py
python preprocessing/validate_cache.py

# 3. (Optional) Build integrated view
python preprocessing/build_integrated.py

# 4. Launch
./start.sh                # Default port 8050
./start.sh -p 8051        # Custom port
```

Open **http://0.0.0.0:8050** in your browser.

## Data Pipeline

The application does not read raw `.h5ad` files at runtime. A preprocessing pipeline converts them into columnar Parquet files with PyArrow's memory-mapped I/O for fast access.

```
04_clustered.h5ad (external directory)
    │
    ├──[build_data_cache.py]──→ data/{dataset_id}/umap_metadata.parquet
    │                           data/{dataset_id}/gene_expression.parquet
    │
    ├──[extract_gene_metadata.py]──→ data/gene_index.json
    │
    └──[build_integrated.py]──→ data/integrated/integrated_umap.parquet
                                data/integrated/integrated_info.json

validate_cache.py ← runs after cache building to check integrity
```

### Preprocessing Scripts

| Script | Input | Output | Notes |
|--------|-------|--------|-------|
| `build_data_cache.py` | `04_clustered.h5ad` per dataset | `data/{id}/*.parquet` | `--data-dir` flag to specify external h5ad directory |
| `extract_gene_metadata.py` | Cached parquets | `data/gene_index.json` | Powers client-side gene autocomplete |
| `build_integrated.py` | All cached parquets | `data/integrated/` | Harmony-PyTorch integration |
| `validate_cache.py` | Cached parquets | Validation report | NaN, bounds, column coverage checks |

## Production Deployment

```bash
# Build image
docker build -t vis-website .

# Run with data volume mount
docker run -p 8050:8050 -v /path/to/data:/app/data vis-website
```

For a full production stack, place nginx in front:

```bash
docker run -d --name vis-app -v /path/to/data:/app/data vis-website
# Then configure nginx with the provided nginx.conf
```

### Docker Image Details

- **Base**: `python:3.11-slim` (required for scanpy/numba compatibility)
- **Runtime**: Gunicorn with 4 workers, 120s timeout (accommodates large UMAP rendering)
- **User**: Non-root `appuser` (UID 1000)
- **Health check**: HTTP probe on port 8050 every 30s
- **Data**: Mount `/app/data` as a volume; data files are not baked into the image

## Key Design Decisions

- **No Plotly Express** — all figures use `plotly.graph_objects.Figure` and `make_subplots` for full control over rendering.
- **Memory-mapped Parquet** — metadata is loaded via `pyarrow.parquet.read_table` with memory mapping. Gene expression columns are loaded on-demand per gene query to avoid loading the full matrix.
- **Adaptive sampling** — when a dataset exceeds 500,000 cells, the UMAP renderer applies uniform downsampling to keep interactions responsive.
- **Client-side gene search** — `gene_index.json` is loaded once per session for instant autocomplete; actual expression values are fetched server-side.
- **Scattergl / Scatter3d** — 2D UMAPs use GPU-accelerated WebGL rendering; 3D views use Plotly's Scatter3d trace type.
- **Module-level caching** — loaded DataFrames are cached in module scope to avoid re-reading across callbacks.

## Development Notes

- **No Node.js / npm / frontend build tools** — pure Python, single `style.css` served by Dash's static asset handling.
- **No tests** currently in the repo. The `validate_cache.py` script provides data integrity checks.
- **Virtual environment** is expected at `../.venv` relative to the repo root.
- **Callback scope**: Callbacks are defined in page modules, not components. Standard Dash callbacks only (no pattern-matching callbacks).
- **.omo/** and **.sisyphus/** directories are agent workspaces and gitignored.

## File Size Notes

- The `data/` directory contains GB-scale Parquet files and is gitignored.
- `gene_index.json` can be large (thousands of genes × multiple datasets) — gene expression queries are server-side filtered in callbacks.
- `dataset_registry.json` holds metadata (species, stage, cell counts, column schemas) and must stay in sync with cached data.
